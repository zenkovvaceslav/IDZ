# Описание работы программы

Программа по таблице нагрузки и списку ставок считает для каждого
преподавателя недогруз — сколько часов ему не хватает до нормы по ставке.

## Входные данные (Что получаем на ввод)

Программа ожидает на вход два набора данных в формате .xlsx (таблицы Excel):

- **Файл нагрузки** — таблица, где каждая строка это одна позиция нагрузки.
  В ней программа ищет столбцы с ФИО преподавателя и с часами (`Нагрузка`).
- **Список ставок** — таблица, где у каждого преподавателя указана его доля
  ставки. Здесь программа ищет столбец с ФИО и столбец со ставкой.

Какие именно файлы, листы и столбцы читать, задаётся в начале функции
main() (например, нагрузка берётся с листа «общее», а ставки — с листа
«Лист2»). Список ставок может лежать как в отдельном файле, так и на другом
листе того же файла.

## Выходные данные (Что получаем на вывод)

Результат программа печатает в консоль и сохраняет в файл. На экран
выводится:

- Сообщение о том, для скольких преподавателей ставка нашлась в списке, а
  скольким проставлена ставка по умолчанию.
- Итоговая таблица со столбцами: Преподаватель, Назначено (суммарные часы),
  Ставки, Норма, Недогруз и Статус (недогружен / перегружен / норма). В
  таблицу попадают только реальные преподаватели — служебные строки
  (дипломы, комиссии, отчисления) и случайные числа в столбце ФИО
  отсеиваются.
- Общая сумма недогруза по всем недогруженным преподавателям.

Кроме того, эта же итоговая таблица сохраняется в отдельный файл
недогруз.xlsx. Исходные файлы программа не меняет.

## Как работает код

Скрипт написан на функциях и использует библиотеку pandas для работы с
таблицами. Код работает в несколько этапов:

**1. Подключение библиотеки**
В начале подключается pandas (import pandas as pd) — это библиотека, которая
позволяет читать таблицы Excel и работать с ними прямо из кода.

**2. Вспомогательные функции**
Перед главным расчётом объявлены пять небольших функций-помощников:
- detect_column — находит нужный столбец в таблице по списку возможных
  названий, без учёта регистра; если столбца нет — выводит понятную ошибку.
- normalize_name — приводит ФИО к единому виду (убирает лишние пробелы,
  меняет ё на е), чтобы один и тот же человек не посчитался дважды.
- is_teacher — проверяет, что строка это настоящее ФИО, а не служебная
  запись. Отсекает значения без русских букв (числа) и строки со словами
  «диплом», «комиссия», «отчисление», «компенсация».
- status_of — по числу недогруза возвращает текст: больше нуля —
  «недогружен», меньше нуля — «перегружен», ноль — «норма».
- rates_to_map — превращает список ставок в словарь вида {ФИО: ставка},
  чтобы потом быстро находить ставку по фамилии.

**3. Главный расчёт**
Функция compute_underload выполняет основную работу. Сначала она определяет,
в каких столбцах лежат ФИО, часы и (если есть) ставка. Затем причёсывает
ФИО, переводит часы в числа и оставляет только настоящих преподавателей с
помощью is_teacher. После этого группирует строки по преподавателю и
суммирует все его часы (метод groupby вместе с sum), при этом отрицательные
строки компенсации корректно вычитаются. Дальше для каждого преподавателя
подставляется ставка — из столбца таблицы, из списка ставок или по
умолчанию. В конце считается норма (ставка × 830), недогруз (норма − часы)
и статус, и возвращается готовая таблица.

**4. Подсчёт общего недогруза**
Функция total_underload складывает недогруз по всем преподавателям, у
которых он положительный, — то есть показывает, сколько всего часов не
хватает.

**5. Сборка**
Функция main — точка входа в программу. Здесь задаются пути к файлам,
названия листов и столбцов. Программа читает нагрузку и список ставок через
pd.read_excel, передаёт их в compute_underload, печатает сообщение о
найденных ставках и саму таблицу, а затем сохраняет результат в Excel
методом to_excel.

**6. Запуск**
Конструкция if __name__ == "__main__" в самом конце нужна, чтобы функция
main() сработала только при прямом запуске этого файла, а не при импорте
его как модуля в другой код.
